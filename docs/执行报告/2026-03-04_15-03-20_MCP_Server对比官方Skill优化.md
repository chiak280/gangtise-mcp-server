## 对比分析结论

| 维度 | 官方 Skill (gangtise-kb) | 我们的 MCP Server |
|------|--------------------------|-------------------|
| 接口覆盖 | 1 个（knowledge/batch） | 6 个工具（知识库/指标/Agent/报告） |

## 修改内容

### 1. 修复 batch 响应解析（`gangtise_client.py:205-238`, `tools/knowledge.py:136-148`）

**问题**：批量接口返回格式是 `[{query: "...", data: [...]}]`（列表），代码按 `dict` 处理。

**修复**：
- `search_knowledge_batch()` 返回类型 `dict` → `list[dict]`
- `handle_knowledge_batch()` 改为先构建 `result_map`，再按 query 匹配

### 2. 添加 knowledgeNames 参数（`gangtise_client.py:24,187,224`）

**问题**：官方 Skill 显式传 `knowledgeNames: ["system_knowledge_doc"]`，我们遗漏。

**修复**：在 `search_knowledge()` 和 `search_knowledge_batch()` 的 payload 中添加该字段。

### 3. 添加 429 重试逻辑（`gangtise_client.py:25-26,155-165`）

**问题**：API 有调用频率限制（HTTP 429），未处理。

**修复**：新增 `_post_with_retry()` 方法，遇 429 自动退避重试（1s/3s/5s，最多 3 次）。知识库查询和指标查询均已改用此方法。
